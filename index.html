PingoTunnel

HTTP over ICMP
using Go on the Pi

Overview

In many venues, such as hotels, airports, and conference centers, one can connect to the local Wifi, and yet cannot access the internet. Although you are given an IP address, in order to browse or otherwise exchange packets over TCP/IP you must first use a browser to login using a supplied password or in some cases provide payment details.

Before logging in however, it is often possible to bypass the paywall using a subset of Internet Protocol (IP) called Internet Control Message Protocol (ICMP), or more specifically, Ping. That is, you cannot exchange TCP/IP packets (browse), but you can use "ping" to reach an external server.

The idea is not mine, I first saw the Ping Tunnel project many years ago. I'm not sure who thought of it first, but there are many implementations. Since first looking into the problem space, two issues have made it feasible for me to attempt this.

The first breakthrough is the introduction of the Raspberry Pi. Instead of needing a large, power-sucking, always-on PC in order to provide the necessary remote server, this inexpensive, power-frugal, wall wart Linux server has all the necessary bits to do the server side task.

The other breakthrough, is the introduction of the Go language. If you've ever programmed on Linux, you know that it is about as much fun as poking a sharp stick in your eye - repeatedly. Although Go is only moderately better than that, it offers two compelling benefits:

  • It is self contained and portable. That is, if you can get Go running on your machine, you have everything that is needed... mostly.
    It's like Java that way, in that the portability issues have been abstracted out for you and you can assume that a binary produced by Go will run without needing to upgrade every library or recompile the kernel on the target machine. This is especially important for execution on the Raspberry Pi and Android phones where the small size won't easily allow that.
  • It is built for networking. This is Google's language of choice for new server installations and they know a bit about networking. Go has a lot of libraries that support networking and implementing this type of program is somewhat easier than in C# or Java land.
